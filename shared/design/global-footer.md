# Page design spec: global header + mega-footer (site-wide components)

Design-only spec. Consumes `shared/page-briefs/global-footer.md`. SEO owns the link set, labels, NAP, and the regulated FMCSA wording; this defines look, layout, hierarchy, components, responsive behavior, and the conversion path. Built on `design-system.md`. Two components, one spec: the sticky global header and the global mega-footer, both identical markup on every page. No em dashes.

## Page role
The two site-wide chrome components every page inherits. The header keeps the dual-silo model legible and the single primary action reachable from any scroll position. The footer makes every money page one click away from anywhere (link equity, crawl discovery), shows the full service breadth the niche expects, and carries NAP plus the one honest trust line. Neither component competes with a page's own primary action; both reinforce it. The home, compliance hub, and trailer specs all reference this file rather than restating the chrome.

## Section order and layout

### A. Global sticky header
1. Single sticky bar, Ink surface (Cloud wordmark and Steel-tinted nav text on it, the system's highest-trust pairing at 16.2:1), full width, content held to the 1200px grid with 24px gutters. One soft hairline (1px Slate at 16%) under the bar only after scroll, so the resting hero reads as one calm field. Height around 64px desktop, 56px mobile, both clearing the 44px hit-target floor.
2. Left: the mono "TECH [amber tick] RIG" wordmark (Cloud-on-Ink variant), linking home, sized at or above the 96px desktop / 80px mobile minimum width with cap-height clear space. The amber register tick is the only Signal mark in the resting header, so it never competes with the page's amber CTA below.
3. Center to right (desktop): the two distinguished menu groups, then About and Contact as plain top-level links.
   - "Compliance" and "Dispatch" are the two mega-menu triggers, visually distinguished from the plain links and from each other so the dual-silo model reads in one glance: give each a small leading single-line icon (a filing/document mark for Compliance, a trailer mark for Dispatch) and a hairline group treatment. Compliance sits first (compliance-led hierarchy). They are siblings in weight, not one dominant, since both silos are real.
   - Compliance mega-panel (on hover and on keyboard focus): a Cloud panel under the bar holding the compliance links in two tidy sub-columns led by the hub link, drawn from the same set as footer Column 1 so they never drift. Dispatch mega-panel: a single column of the six trailer links led by the hub link, each with its trailer line icon matching the icon used on that trailer's hub card.
4. Right edge: the persistent conversion pair, in the system CTA hierarchy.
   - Primary: one Signal amber button with Ink text carrying the current page's single primary action (label is the page's own CTA wording, supplied by that page's spec and SEO, for example "Start your compliance setup" on home and the compliance hub, "Get my box truck dispatched" on the box-truck page). This is the header's only Signal element.
   - Secondary: a Call button as an Ink outline (or Cloud outline on the Ink bar) with a small phone line icon, `tel:+19179092257`, text "Call". Subordinate to the amber primary, never amber itself.
5. Mobile header: Cloud-on-Ink wordmark left; a drawer toggle right (44px target). The amber primary CTA and the Call button stay visible in the bar at all widths (they are the conversion floor, never hidden behind the drawer). The drawer is a full-height Ink panel: the two labeled groups "Compliance" and "Dispatch" as collapsible accordion sections (chevron rotates, respects reduced motion), each listing the same links as its mega-panel, then About and Contact. The same amber primary and Call repeat at the foot of the drawer for thumb reach.

### B. Global mega-footer
6. Trust + NAP band (top of the footer, full-bleed Ink, spanning above the link columns): the system reserves Ink for high-emphasis moments and the footer is one. Left: the stacked logo lockup (Cloud-on-Ink wordmark plus the optional mono descriptor line). Beneath it the one-line positioning "Trucking compliance and dispatch, from your first filing to your next load." as a styled Plex Sans paragraph, never a heading. Right or below (responsive): the NAP set in the system's mono "official record" treatment for the address and contacts: 30 N Gould St, Ste R, Sheridan, WY 82801, then `tel:+19179092257` and `mailto:info@techrig.org` as linked lines. Address presented as the registered address, no "visit us" or hours, no map embed (it is not a walk-in office and a map would imply one plus cost CWV).
7. The four navigation columns (Cloud text on the Ink field, or a single half-tone Ink panel under the band; keep it flat, no card chrome). Equal-top alignment, generous column gutters, each column headed by an Archivo H-less label set as a styled paragraph (footer headings are navigation labels, not document headings, so they are not H-tags). Column link labels are the descriptive service names from the brief (footer links are nav, so the "no click here / no our services" inline rule does not apply here).
   - Column 1, "Compliance Services": all 15 links from the brief in the brief's build/priority order, hub and Authority Package first (Compliance Services hub, Authority Package (DOT + MC), USDOT Registration, MC Authority, BOC-3 Filing, Insurance Filing, UCR Registration, MCS-150 Update, IRP Registration, IFTA Registration, Driver Qualification Files, Drug & Alcohol Consortium, FMCSA Clearinghouse, ELD Services, Trucking LLC). Because this is long, split it into two sub-columns under the single "Compliance Services" header so it balances the others, the hub link reading first at the top-left of the pair.
   - Column 2, "Dispatch Services": the hub link then the six trailer pages (Truck Dispatch hub, Box Truck Dispatch, Reefer Dispatch, Flatbed Dispatch, Dry Van Dispatch, Power Only Dispatch, Hot Shot Dispatch). Each trailer label may carry the same small trailer line icon used in nav and on its hub card, kept quiet so the column stays scannable.
   - Column 3, "Company": About Tech Rig, Contact, How to Start a Trucking Company (the pillar), New Authority Loads, Blog.
   - Column 4, "Legal": Terms of Service, Privacy Policy, Power of Attorney, Refund Policy, Referral Program.
8. The FMCSA credibility line, set apart as a worded line beneath or beside the columns, exact text: "DGR Tech Rig LLC is officially listed by FMCSA as a BOC-3 blanket process-agent company." Treated as a sentence in body/mono, never as a badge, seal, shield, or anything implying government endorsement. This is the footer's only trust claim.
9. Social slot: a single quiet row of CSS or inline-SVG line icons for Facebook, Instagram, Twitter/X, Google Business, LinkedIn. URLs are PENDING from the client: render only the icons whose URL is supplied and omit the rest entirely (no greyed placeholders, no links to "#"). The row must read fine if it is empty today (the slot collapses cleanly), and read fine when one to five icons arrive later. Icons are monochrome line marks on the Ink field, not brand-colored tiles.
10. Legal baseline strip (bottom): "© [current year] DGR Tech Rig LLC. All rights reserved." in Slate-on-Ink small text, year rendered dynamically by Dev. Legal entity name appears here (and only here plus legal pages and schema); public brand "Tech Rig" everywhere else.

## Hierarchy and the visual path
Header: the eye reads wordmark, then the two labeled silo groups, then lands on the single amber primary CTA at the right edge, the only Signal element in the bar; the Call button sits just inside it as the clear secondary. The header never introduces a second amber element, so it cannot dilute the page's rationed Signal action. Footer: the Ink band's positioning line and NAP read first as "who and where," then the eye drops into the four columns as a deliberate site index (breadth signal), then the single FMCSA line as the one trust note, then the legal baseline. Nothing in the footer is amber: the funnel CTAs live in page bodies, the footer is navigation and trust, so Signal stays rationed sitewide.

## Imagery and illustration
No photography. Header: the mono wordmark, two small single-line silo icons (filing mark, trailer mark), the trailer line icons in the Dispatch panel, a phone line icon on Call, a chevron on mobile group toggles. Footer: the stacked wordmark lockup, optional small trailer icons on Column 2 labels, the social line icons. Everything is inline SVG or CSS at 2px stroke from the system icon language, deliberately lightweight: the brief calls for a text-and-CSS footer with no heavy assets to protect the weak CWV baseline, so no images, no map tiles, no icon fonts.

## Motion
Minimal and state-clarifying only. Header: the under-bar hairline appears on scroll; mega-panels open with a calm fade or short slide (no more than 200ms, ease-out); the mobile drawer slides in and its group chevrons rotate. Hover and focus give micro-feedback on links and buttons. Banned here as everywhere: scroll-triggered fades on the footer columns, anything auto-playing, anything that moves while the user reads. `prefers-reduced-motion: reduce` drops every header and drawer transition to instant final states (panels simply appear, drawer simply shows).

## CRO treatment
- The header carries the page's single primary action and a persistent Call at every scroll depth, so the conversion path is never more than one glance away; the footer deliberately carries none, keeping Signal rationed to the page body.
- The footer is a conversion asset through reach, not buttons: every money page is one click from any page, and the full breadth reassures a stressed buyer that the team covers the whole journey.
- Trust is built only from the real: the exact FMCSA listing line and the verifiable NAP. No metrics, no stars, no ratings, no awards, no trust badges, no guarantees anywhere in the chrome (brand discipline applies to global components too).
- The Call button is a friction-removal device for a buyer who often wants to talk before filing; it uses a real `tel:` and stays reachable on mobile alongside the primary CTA.
- Identical markup on every page keeps the internal-link graph and the conversion floor consistent sitewide.

## OG image
Not applicable: these are site-wide components, not a routable page, so they own no OG image. Per-page OG images are generated by each page's own spec from the system template. The footer's stacked logo lockup is the same wordmark treatment those OG images use, kept visually consistent.

## What Dev must preserve
- The exact link set, labels, and destinations from the brief; every footer and header link must resolve to a URL in `sitemap-plan.md`, and no `noindex`/transactional URL (`/get-started`, `/single-payment`, the form subdomains) may appear in either component.
- Identical header and footer markup on every page (consistent link equity); the legal entity name "DGR Tech Rig LLC" only in the footer trust line, FMCSA line, and copyright, with "Tech Rig" as the brand everywhere else.
- The FMCSA process-agent line worded exactly as the brief states, treated as a sentence, never a badge or seal, never implying government affiliation or endorsement.
- The amber-never-white-text rule and exactly one Signal element in the header (the primary CTA); the footer carries no Signal element.
- The pending-social discipline: omit any social icon whose URL is not supplied, never link to a placeholder or "#"; the slot degrades gracefully empty.
- NAP exact (registered address, no implied walk-in office or hours), real `tel:` and `mailto:`, address and contacts in the mono treatment; year in the copyright rendered dynamically.
- AA contrast on the Ink field, visible keyboard focus on every link and control, full keyboard operability of the mega-menus and the mobile drawer, hit targets at least 44px, and reduced-motion honored.
- Lightweight by mandate: text links and CSS/inline-SVG icons only, no map embed, no icon fonts, no heavy assets in the chrome.
