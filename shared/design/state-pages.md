# Page design spec: /tech-rig-dispatch-[state]/ (state pages template: TX, CA, NY, FL + future)

Design-only spec. Consumes `shared/page-briefs/state-pages.md`. SEO owns copy, headings, the per-state primary/secondaries, CTA wording and destinations, and which state facts are verified; this defines look, layout, hierarchy, components, imagery, motion, and the conversion path. Built on `design-system.md`. This is one shared template for the four existing realigned pages (Texas, California, New York, Florida) and every future `/tech-rig-dispatch-[state]/`, with the per-state deltas called out so no two states look or read alike where the brief forbids it. No em dashes.

## Page role
A realigned, traffic-bearing state page. These URLs already rank and are kept; the realign shifts intent toward state-level setup and authority (the real client-origin play) while keeping a genuine dispatch section. One job per visit: a carrier in this state understands "this team gets a trucking company road-legal here, then keeps it loaded," is routed to the national money pages that do the filing, and can still reach dispatch. Compliance-led order, mixed intent (MOFU to BOFU). The page adds only the state-specific layer; it never duplicates national money-page content, and it never competes with those pages for the filing itself. State pages are the site's top duplication risk, so the template is built to force per-state difference, not absorb it.

## Section order and layout

1. Header (global): per `global-footer.md`. The persistent primary CTA carries this page's setup action (its wording is SEO's, for example "Start your [State] setup" or the shared compliance-setup CTA), routing into the compliance path the brief specifies.

2. Hero (Paper, asymmetric two-column; copy-first stack on mobile)
   - Left (dominant): H1 with the state primary, for example "How to Start a Trucking Company in Texas" (Archivo, exact SEO wording). The brief's state-specific hero lede as a styled Plex Sans Body L paragraph, never an H-tag. This lede is the first and highest-risk uniqueness surface: each state leads with a different state fact (Texas intrastate-vs-interstate nuance; California regulatory environment; New York Northeast/HUT context; Florida port/import angle), so the template treats the lede as a per-state slot, never shared boilerplate. Then the primary button (Signal amber, Ink text) into the setup path, and a subordinate Steel text link "or see what we file" that scroll-anchors to the filings section.
   - Right (signature visual): a scoped Authority Status Tracker (Application filed -> 21-day federal protest period -> Authority active), the same honest stepper the compliance pages use, here framing the setup journey a carrier in this state is about to start. Honesty rules apply: the 21-day step is the fixed federal protest period, never a Tech-Rig-controlled countdown or guaranteed activation date. The tracker is state-agnostic in content (federal sequence is the same everywhere), so it provides a consistent anchor while the copy carries the state difference. No state-shaped map art and no state outline used as a credibility prop.
   - 5-second test: the H1 names the state and the goal, the lede names a real state specific, the tracker shows the federal setup path resolving to active authority. The amber CTA is the single Signal element above the fold.

3. H2 "Starting a trucking company in [State]" (Cloud surface): the state-nuance beat, the page's primary reason to exist. This is a per-state slot by mandate (no shared 8+ word sentences across states). Lay it out as a typographic explainer, not a card grid: the brief's state copy on a readable measure (around 68ch), with the genuine state distinctions (intrastate vs interstate operation, any state-level registration or state DOT, state fuel/road tax context) called out as a short Slate-toned definition list or inline emphasized terms, not invented stat callouts. Where the brief supplies a verified state figure, render it once in the mono "official record" treatment as a single quiet stat; where no verified figure exists, the layout omits the stat cleanly and leans on the explainer copy. The design must read complete with zero stats present (the old fabricated "$1.6 trillion" type claim is removed, and nothing in the layout begs for a number to fill).

4. H2 "What we file for [State] carriers": the bridge from state context to the national filing pages, where the actual conversion lives. Render as the system's service-card grid (the same content-bearing card component as the compliance hub: small line icon, service name as a Steel link, one plain line, price chip from the single source), but as a curated subset routing UP to the national money pages the brief names (USDOT, MC authority, the full compliance hub, and the bundle), not a duplicate of the hub's full grid. Each card links to its national page (`/dot-registration/`, `/mc-registration/`, `/compliance-services/`, `/mc-dot-registration/`, and the others the brief lists), making explicit that the filing is done on the national page and the state page is the on-ramp. Price chips render from the single source with service-fee-vs-government-fee separation intact; no state page hardcodes or restates a price.

5. H2 "Truck dispatch in [State]": the retained dispatch beat (these pages were originally dispatch-oriented, and the realign keeps dispatch relevance). Deliberately lighter than the setup sections above it and visually distinct via Steel accents (the system's dispatch-silo cue), so the compliance-led hierarchy is unmistakable while dispatch stays present. A short copy block plus a quiet capability strip (the six trailer line icons, or the subset relevant to the state if the brief scopes it) and an inline 1 to 3 word anchor "truck dispatch" into `/services/`. The dispatch percentage model, if shown, uses the mono price treatment from the single source. No second primary button competes here; the dispatch link is Steel and subordinate.

6. H2 funnel bridge (optional full-bleed Ink band, used when the page warrants the high-emphasis beat): the one-team handoff, a simple two-node line diagram (Get road-legal -> Keep loaded) tying the setup and dispatch sections together, with the pillar link "starting a trucking company" -> `/how-to-start-a-trucking-company/` and `/services/` as the per-brief state-page links up to the national pillar and dispatch hub. Reserved Ink, used once, not on every band.

7. H2 "[State] trucking FAQ": the system FAQ accordion (FAQPage schema), rendered only when a real, state-specific FAQ exists per the brief. Each row is one question; the activation-timing answer, if present, is an honest-expectations moment paired in tone with the tracker (no guaranteed dates). When a state has no genuine FAQ yet, this section is omitted entirely rather than padded with generic national questions (duplication and thin-content discipline).

8. Closing CTA band: one styled lead line plus the single dominant setup CTA (Signal amber, Ink text), the same destination as the hero. One clear action per page.

9. Mega-footer (global, see `global-footer.md`): the complete national service lists live here, so the state page body stays a curated on-ramp.

## Per-state deltas (what changes between states, what stays fixed)
- Fixed across all states: the section order, the component set, the scoped Authority Status Tracker content (federal sequence), the service-card component and its single-source price chips, the dispatch strip, the global header and footer, the motion and accessibility rules.
- Per-state slots, written independently for each state (no shared 8+ word strings): the H1 state primary; the hero lede and its lead state fact; the entire "Starting a trucking company in [State]" explainer and its state distinctions; any verified state stat (present or cleanly absent); a different worked example or local detail per state; the state FAQ (or its omission); the OG image title and state tag. The four live pages each anchor on the brief's distinct lead (TX intrastate/interstate and in-state freight base; CA regulatory/CARB context and the real CA box-truck client anchor; NY Northeast/HUT; FL port/import), and any unverified figure is omitted, not invented.
- Future states: the template is applied only when a real state-specific layer exists (the documented client-origin play). It is never mass-generated with thin or templated state copy, since that thinness is exactly what caused the original CTR problem. A future state page with no genuine state layer should not ship.

## Hierarchy and the visual path
Eye path: H1 (state goal) -> primary amber setup CTA -> the state-nuance explainer (the reason this page exists) -> the "what we file" cards routing up to the national money pages (the conversion handoff) -> the lighter Steel dispatch beat -> optional one-team bridge -> state FAQ -> single closing CTA. Signal amber is rationed to the one setup action (hero, close, persistent header); every service-card link, the dispatch link, and the pillar anchor are Steel and subordinate. The dispatch section is intentionally visually lighter and Steel-accented so the compliance-led realign reads at a glance. One primary action per view.

## Imagery and illustration
No photography, and specifically no state-outline map art or flag motifs used as decoration or as a credibility prop. Hero: the scoped Authority Status Tracker. Setup explainer: at most one or two single-line icons for state distinctions (an intrastate/interstate route node, a document/filing mark), used sparingly. Filing section: one line icon per service card, matching the national hub's icons so the on-ramp reads consistently. Dispatch strip: the trailer line icons from the system set. Funnel band: the two-node Get road-legal -> Keep loaded diagram. All single-line SVG, 2px stroke, lightweight to protect the CWV baseline these traffic-bearing pages must not regress.

## Motion
Minimal. The hero tracker reveals its steps with a calm staggered fade (no more than 200ms each) on load; nothing else animates by default. Accordion chevron rotates on the FAQ. Sticky mobile CTA appears after the hero scrolls past. No per-section scroll-fade. `prefers-reduced-motion: reduce` gives the tracker its final static state and drops all transitions to instant.

## CRO treatment
- One dominant action per state page (start setup), Signal amber, repeated at hero and close and held in the persistent header; dispatch is the consistent Steel secondary, present but subordinate per the realign.
- The conversion mechanic is the routing handoff: the state page earns trust with a genuine local layer, then sends the carrier to the national money page that actually files, so equity concentrates on those pages and the state page does not cannibalize them.
- Honest expectations up front via the tracker (fixed federal protest period, no guaranteed dates) reduce the activation anxiety that drives bounce.
- Trust is built only from the documented and the verified: real state distinctions and any sourced state figure, never a fabricated market statistic; the layout reads complete with no stat present, so the design never pressures copy to invent one.
- Mobile: sticky bottom bar with the setup CTA plus a `tel:` Call button; copy-first hero stack; trailer icons wrap; the state explainer stays on a readable measure.

## OG image
Unique branded OG per state from the system template: Ink or Paper field, mono wordmark, title set to the state goal (for example "Start a Trucking Company in Texas"), a category tag in Plex Mono ("Compliance" or a state tag), one relevant line icon, a Signal rule. One template, a distinct variant per state by title and tag; never a placeholder, never a shared image across states.

## What Dev must preserve
- The kept URLs (`/tech-rig-dispatch-texas/`, `-california/`, `-new-york/`, `-florida/`, and the `[state]` pattern); 301 only if a slug must change, logged in `migration-plan.md`. These pages carry traffic; do not drop ranking headings or terms.
- SEO copy, the heading structure, the per-state primaries and secondaries, and every internal link and destination exactly. The hero lede and any bold lead-ins are styled paragraphs, never H-tags.
- Per-state uniqueness: no shared 8+ word sentences across state pages; each lede, state-nuance section, and FAQ is independent. Run the duplicate-string scan across all state pages before launch.
- No fabricated state market statistics; render a stat only where the brief marks it verified, and degrade gracefully (omit cleanly) when absent. No guaranteed activation timelines anywhere; the protest period is the fixed federal step.
- Reuse, do not duplicate, the national money pages: the state "what we file" cards link up to them and never restate their content; price chips come from the single source with service-fee-vs-government-fee separation, no price hardcoded.
- The Authority Status Tracker honesty rules (federal sequence, no guaranteed dates); the scoped tracker content is identical across states.
- Amber-never-white-text; one Signal action per view; the dispatch section kept visually subordinate (Steel) to the compliance-led order; AA contrast; visible keyboard focus; reduced motion; hit targets at least 44px.
- Documented proof only; no ratings, no invented testimonials, no performance percentages. Decorative glyphs and check marks via inline SVG or CSS, never typed into headings.
